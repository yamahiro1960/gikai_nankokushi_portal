議案PDF登録

ローカルに保存した議案資料を確認し、公開用コピーを Supabase へアップロードする画面です。ローカルの原本ファイルはそのまま残ります。

モード: 原本確認 + 公開アップロード

保存先フォルダー

議案PDFの原本は、ポータルの作業フォルダー直下に作成した gian フォルダー配下へ、フォルダー名ルールに沿って配置します。

ローカル作業フォルダー: C:\Users\nanko\OneDrive\デスクトップ\My_Developments\giin\gikai_nankokushi_portal

フォルダー名ルール: 回次_西暦_月

例: 425_2022_03

意味: 425=議会回数 / 2022=西暦年 / 03=月

配置先の基本形: gian\425_2022_03\PDFファイル名.pdf

勉強会資料の配置先: gian\425_2022_03\benkyoukai\PDFファイル名.pdf

運用: このフォルダーは原本保管用として残し、Web から Supabase へ公開用コピーをアップロードします。

この画面の役割

1. ローカルの gian フォルダーに置いた原本ファイルを確認します。

2. フォルダー単位で Supabase Storage に公開用コピーをアップロードします。

3. アップロード後も、ローカルの原本ファイルは削除されず、そのまま残ります。

入れるファイルの種類

現在の画面では、次の5種類のPDFを想定しています。既定のファイル名は変えずにそのまま配置してください。

議案

例: 議案第１号：令和７年度一般会計補正予算.pdf

報告

例: 報告第１号：一般会計補正予算.pdf

提案理由

例: 提案理由（446回）.pdf

施政方針

例: 施政方針（446回）.pdf

勉強会資料

例: gian\425_2022_03\benkyoukai\R8.2.24議員勉強会資料（当初予算）.pdf

作業手順

  1. 1

    デスクトップ内に議案フォルダーを作成

    例: gian

  2. 2

    フォルダー名を「回次_西暦_月」の形式で作成

    例: 446_2026_03（第446回議会、2026年3月）

    形式: {回次}_{西暦}_{月}

  3. 3

    PDFファイルをフォルダーに配置

    勉強会資料は benkyoukai サブフォルダーに配置

  4. 4

    下の「フォルダー選択」ボタンで作成したフォルダーを指定

    内容確認後に「実行」をクリックすると、Supabase へ公開用コピーのアップロードが開始します
